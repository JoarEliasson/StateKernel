# StateKernel Brand Assets

This directory contains the canonical brand assets for StateKernel.

StateKernel is an industrial-modern, local-first engineering product.  
The brand system should feel precise, technical, measured, and visually clean.

## Files

### `statekernel-logo-primary.png`
Primary brand mark.

Use this as the default StateKernel logo for:

- repository README
- architecture docs
- public documentation
- project pages
- general brand references

This is the canonical everyday logo.

### `statekernel-logo-premium-hero.png`
Premium hero variant.

Use this for:

- portfolio hero sections
- presentation covers
- case-study headers
- social preview graphics
- polished showcase surfaces

This variant is intended for larger-format use where a slightly more premium, display-oriented version is appropriate.

### `statekernel-logo-icon.png`
Icon-optimized variant.

Use this for:

- GitHub avatar
- favicon
- app icon
- navbar logo
- small-size UI usage
- future desktop app packaging icons

This is the preferred small-scale mark.

## Usage rules

Use the primary logo by default.

Use the icon logo for small-size or system-level surfaces.

Use the premium hero logo only when the surface is presentation-oriented and benefits from a more display-focused version.

Do not treat all logo variants as interchangeable.

## Naming convention

These filenames are the canonical filenames for current raster assets:

- `statekernel-logo-primary.png`
- `statekernel-logo-premium-hero.png`
- `statekernel-logo-icon.png`

If vector versions are added later, place them under `source/` and preserve the same naming convention.

Suggested future filenames:

- `source/statekernel-logo-primary.svg`
- `source/statekernel-logo-premium-hero.svg`
- `source/statekernel-logo-icon.svg`

## Color intent

The logo system aligns with the StateKernel brand palette:

- dark structural body
- light outline
- signal blue
- process teal
- pulse violet
- success green

Do not arbitrarily recolor logo variants without updating the brand documentation.

## Background guidance

Preferred usage:

- dark backgrounds
- clean neutral surfaces
- high-contrast technical layouts

Use caution on:

- noisy photographic backgrounds
- highly saturated backgrounds
- surfaces where the light outline loses clarity

## Editing guidance

Do not stretch, crop, recolor, or rearrange the logo geometry.

If a new format or export is needed:

- derive it from the highest-quality source
- preserve spacing and proportions
- keep naming consistent
- document the new variant if it becomes an approved asset

## Repository guidance

This folder is the canonical committed location for public brand assets.

Internal exploration files, rough drafts, and discarded variants should not be stored here unless placed in a clearly marked archive subfolder.
