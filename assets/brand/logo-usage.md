# Logo Usage

## Purpose

This document defines how StateKernel logo variants should be used across the repository, documentation, UI surfaces, and future product materials.

The goal is consistency, clarity, and a professional public presentation.

## Approved logo variants

StateKernel currently uses three approved logo variants:

### 1. Primary logo
File:
- `assets/brand/statekernel-logo-primary.png`

Role:
- canonical everyday logo

Recommended uses:
- repository README
- public documentation
- architecture documents
- case-study content
- general product branding

This should be treated as the default logo unless a stronger reason exists to use another approved variant.

---

### 2. Premium hero logo
File:
- `assets/brand/statekernel-logo-premium-hero.png`

Role:
- large-format showcase variant

Recommended uses:
- portfolio hero sections
- slide covers
- polished presentation surfaces
- social preview graphics
- long-form project feature pages

This variant should be used selectively.
It is not the default logo for routine documentation or UI chrome.

---

### 3. Icon logo
File:
- `assets/brand/statekernel-logo-icon.png`

Role:
- small-size and system-level mark

Recommended uses:
- GitHub avatar
- favicon
- app icon
- navbar corner logo
- small UI placements
- future desktop packaging assets

This is the preferred variant whenever the logo must remain readable at small sizes.

## Usage hierarchy

When choosing a logo variant, follow this order:

1. use the primary logo by default
2. use the icon logo for small surfaces
3. use the premium hero logo only for presentation-oriented display contexts

## Do not

Do not:

- use variants interchangeably without reason
- recolor the logo arbitrarily
- distort proportions
- crop too tightly
- place the logo on visually noisy backgrounds
- use low-contrast surfaces where the outline loses clarity
- create unofficial derived variants without documenting them

## Background rules

Preferred backgrounds:

- obsidian
- graphite
- steel
- other dark neutral brand surfaces
- clean light neutral surfaces where contrast remains strong

Use caution on:

- saturated gradients
- highly textured images
- bright photographs
- surfaces that compete with the colored nodes

## Spacing guidance

Always preserve clear space around the logo.

As a practical rule, maintain at least the diameter of one small node or equivalent visual padding around the mark where possible.

Do not crowd the logo with adjacent text, icons, or panel borders.

## Sizing guidance

### Large usage
Use:
- primary logo
- premium hero logo when appropriate

### Medium usage
Use:
- primary logo

### Small usage
Use:
- icon logo

If the logo starts to lose clarity, switch to the icon variant instead of shrinking the primary mark too aggressively.

## Repository usage recommendations

### In the repo
Use:
- primary logo in `README.md`
- icon logo for GitHub/profile-style surfaces if needed
- premium hero logo only in curated presentation or case-study materials

### In docs
Use:
- primary logo by default

### In UI
Use:
- icon logo for navbar/app chrome
- primary logo for onboarding or splash-style larger placements

## Future asset guidance

If new logo formats are added later, they should:

- follow the existing naming convention
- be added under `assets/brand/`
- use `assets/brand/source/` for vector/source versions
- be reflected in this document if approved for ongoing usage

## Decision summary

StateKernel logo usage should follow this model:

- **Primary logo** = default brand mark
- **Premium hero logo** = display/showcase variant
- **Icon logo** = small-size/system variant

This keeps the identity consistent while still allowing purpose-specific flexibility.
